--------------------------------------------------------------------------------
PROJECT OVERVIEW
--------------------------------------------------------------------------------
Flutter + Android (Kotlin) indoor SLAM mapping and navigation app.
- Flutter side: map visualization, UI
- Kotlin (Android) side: ARCore SLAM, YOLOv11 object detection, navigation
- Communication: MethodChannel (Platform Channel)

--------------------------------------------------------------------------------
KEY DEPENDENCIES / TECH
--------------------------------------------------------------------------------
- ARCore (Shared Camera mode) — pose tracking, plane detection
- TFLite (YOLOv11) — indoor object detection (model: indoor_nav_best_float16.tflite)
- Android SpeechRecognizer — voice input (no internet required)
- Android TextToSpeech — voice output (no internet required)
- Flutter: minimal deps (cupertino_icons only)
- Grid resolution: RES = 0.20f (20 cm per cell)
- MethodChannels:
    "com.ketan.slam/ar"  — map data (Kotlin→Flutter), openAR (Flutter→Kotlin via MainActivity)
    "com.ketan.slam/nav" — navigation commands (Flutter→Kotlin) + state/instruction (Kotlin→Flutter)

--------------------------------------------------------------------------------
FILE STRUCTURE — Kotlin (android/app/src/main/kotlin/com/ketan/slam/)
--------------------------------------------------------------------------------
MainActivity.kt         — Flutter entry point; launches ArActivity via "openAR" method channel
ArActivity.kt           — Core AR activity; SLAM loop, YOLO, occupancy grid, navigation tick
  - buildLayout()       — adds micButton (🎤/⏹) + navView (instruction banner) to AR view
  - updateSlam()        — computes heading, calls NavigationManager.tick() every GL frame
  - buildMapPayload()   — sends occupancy grid + objects + navPath to Flutter every 800ms
  - updateNavHud()      — updates native nav banner + mirrors state to Flutter navChannel
  - onMicTapped()       — starts voice nav or stops current navigation

BackgroundRenderer.kt   — OpenGL ES2 camera background renderer
DisplayRotationHelper.kt — ARCore display rotation helper
DetectionOverlayView.kt — Native YOLO bounding box overlay (used in ArActivity)
SlamEngine.kt           — Pose history, occupancy grid edges, statistics
SemanticMapManager.kt   — Spatial grid of detected SemanticObjects (ConcurrentHashMap)
SemanticObject.kt       — Data classes: Point3D, BoundingBox2D, ObjectType, SemanticObject, GridCell
YoloDetector.kt         — TFLite YOLO inference + DetectionConfirmationGate

[NAVIGATION MODULE — added 2026-03-02]
VoiceCommandProcessor.kt — Android SpeechRecognizer wrapper + rule-based NLP intent parser
  - parseIntent(text)   — extracts ObjectType + DestinationQualifier from transcript
  - KEYWORD_MAP         — maps keyword lists → ObjectType (door, lift, chair, etc.)

PathPlanner.kt          — A* path planner on the occupancy grid
  - planPath()          — A* with 8-dir movement, obstacle inflation (INFLATE=1 cell), corner-cut prevention
  - smoothPath()        — Line-of-sight string-pull pass to reduce waypoint count
  - inflateObstacles()  — expands WALL/OBSTACLE cells by 1 cell for safety margin
  - octile()            — admissible heuristic for 8-directional A*

NavigationGuide.kt      — TTS + turn-by-turn instruction generator
  - computeInstruction() — heading diff → TurnDirection → instruction text
  - isArrived()         — true when user < 1.0m from goal
  - speak()             — TTS with QUEUE_FLUSH

NavigationManager.kt    — State machine orchestrating the full pipeline
  - tick()              — called every GL frame; resolves intent, checks arrival/deviation, emits instructions
  - resolveIntent()     — destination selection (≥2 observations) + A* planning
  - selectDestination() — filters SemanticMap by type + qualifier (nearest/farthest/left/right)
  - rePlan()            — re-runs A* when user deviates > 2.0m from path
  - needsGrid           — optimization flag: skip grid HashMap copy when IDLE

--------------------------------------------------------------------------------
METHOD CHANNEL PROTOCOL
--------------------------------------------------------------------------------
Kotlin → Flutter (com.ketan.slam/ar):
  "onUpdate"    { position_x, position_y, position_z, heading, edges_count,
                  cells_count, total_objects, chairs, doors, fire_extinguishers,
                  lift_gates, notice_boards, trash_cans, water_purifiers, windows }
  "updateMap"   { occupancyGrid: ByteArray, gridWidth, gridHeight, gridResolution,
                  originX, originZ, robotGridX, robotGridZ, objects: List<Map>,
                  navPath: List<{x,z}> }  ← navPath added by navigation module

Kotlin → Flutter (com.ketan.slam/nav):
  "navStateChange"  { state: String, message: String }
                    states: IDLE, LISTENING, PLANNING, NAVIGATING, ARRIVED, ERROR
  "navInstruction"  { text: String, turn: String, distanceMetres: Double }

Flutter → Kotlin (com.ketan.slam/ar):
  "openAR"          — launches ArActivity (handled by MainActivity)

Flutter → Kotlin (com.ketan.slam/nav):
  "startVoiceNav"   — starts SpeechRecognizer (requests RECORD_AUDIO if needed)
  "stopNavigation"  — stops active navigation

--------------------------------------------------------------------------------
DETECTED OBJECT TYPES (YOLOv11 — 8 classes)
--------------------------------------------------------------------------------
CHAIR, DOOR, FIRE_EXTINGUISHER, LIFT_GATE, NOTICE_BOARD, TRASH_CAN, WATER_PURIFIER, WINDOW
(+ UNKNOWN fallback)
Model file: assets/indoor_nav_best_float16.tflite

--------------------------------------------------------------------------------
CELL TYPE CONSTANTS (mirrored in both Kotlin and Dart)
--------------------------------------------------------------------------------
0 = CELL_UNKNOWN
1 = CELL_FREE
2 = CELL_OBSTACLE  (detected object footprint)
3 = CELL_WALL      (vertical plane projection)
4 = CELL_VISITED   (camera has been here)

--------------------------------------------------------------------------------
NAVIGATION STATE MACHINE
--------------------------------------------------------------------------------
IDLE
  │  user taps 🎤 → startVoiceNav
  ▼
LISTENING  (SpeechRecognizer active)
  │  transcript received → parseIntent()
  ▼
PLANNING   (pendingIntent set, waiting for next tick())
  │  tick() called with grid + semanticMap
  │  → selectDestination() + planPath()
  ▼
NAVIGATING (path active)
  │  every 3s: computeInstruction() → TTS
  │  on deviation > 2m: rePlan()
  │  on arrival < 1m: → ARRIVED
  ▼
ARRIVED / ERROR
